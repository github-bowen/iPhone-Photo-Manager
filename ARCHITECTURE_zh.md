# 架构与原理说明

*其他语言版本: [English](ARCHITECTURE.md), [中文](ARCHITECTURE_zh.md).*

本文档详细说明 iPhone Photo Manager 的技术架构、数据流、各模块设计原理以及关键实现细节。

### 1. 总体架构

```text
┌─────────────────────────────────────────────────────┐
│                    Browser (前端)                     │
│  index.html + index.css + index.js                   │
│  ─ 纯静态文件，由 FastAPI StaticFiles 托管            │
│  ─ Vanilla JS，无框架依赖                             │
└────────────────────┬────────────────────────────────┘
                     │ HTTP REST API
                     ▼
┌─────────────────────────────────────────────────────┐
│               FastAPI Server (后端)                   │
│  server/app.py                                       │
│  ├── API 路由层  (GET /api/photos, etc.)              │
│  ├── 后台任务引擎 (asyncio.create_task)               │
│  └── 安全中间件  (CSP, X-Frame-Options, etc.)         │
├──────────┬──────────┬──────────┬─────────────────────┤
│ scanner  │ thumbnail│ geocoder │ database            │
│ .py      │ .py      │ .py      │ .py                 │
└──────────┴──────────┴──────────┴─────────────────────┘
                     │
                     ▼
┌─────────────────────────────────────────────────────┐
│                  数据存储层                            │
│  data/photos.db     — SQLite（异步 aiosqlite）        │
│  data/thumbnails/   — WebP 缩略图与 JPEG 高清渲染缓存 │
│  photos/            — 原始照片文件（只读）              │
└─────────────────────────────────────────────────────┘
```

**设计原则**：
1. **本地优先** — 所有处理在本机完成，照片数据不离开设备。
2. **零配置启动** — 将照片放入 `photos/` 目录即可，无需预处理。
3. **增量设计** — 只处理新增/删除的文件，避免重复计算。
4. **离线能力** — 地理编码使用离线引擎，不依赖外部 API。

### 2. 启动与数据流

#### 2.1 启动流程
`app.py` 启动时初始化 SQLite schema。根据数据库记录数和环境变量 `SCAN_ON_STARTUP` 决定是进行增量全盘扫描，还是仅处理遗留的后台任务。

#### 2.2 全量扫描流程
扫描分为三个阶段，前端通过轮询实时展示进度：
1. **文件索引**：计算磁盘文件路径集合与数据库路径集合的差集，执行插入或删除。
2. **缩略图生成**：并发调用生成 WebP 缩略图（处理 EXIF 旋转方向）。
3. **地理编码**：通过离线引擎批量解析 GPS 坐标。

#### 2.3 高清原图渲染 (HEIC 处理)
当用户请求 `/api/photos/{id}/render` 时：
- 普通 JPG/PNG 格式直接串流返回。
- HEIC 格式则通过 `pillow_heif` 读取，并将其安全缩放至 4K 级别分辨率（防止爆显存，同时保持满画质视觉体验），转换并压缩为 JPEG 格式后存入本地磁盘缓存。第二次请求时实现 0 延迟秒开。

### 3. 模块详解

#### 3.1 `scanner.py` — 文件扫描与元数据提取
从 HEIC/JPG/PNG 提取 EXIF 数据。不依赖 ffmpeg，直接解析 QuickTime (MOV) 的二进制原子结构提取时长与定位。自动在同级目录寻找同名 `.MOV` 以识别 Live Photo。

#### 3.2 `thumbnail.py` — 缩略图生成
在缓存目录中镜像原始照片的层级结构，使用 Pillow 生成高质量、高压缩率的 WebP 缩略图。

#### 3.3 `geocoder.py` — 离线地理编码
基于 KD-Tree 的离线反向地理编码，将 GPS 坐标四舍五入到 2 位小数（约 1.1km 精度）后聚类去重，实现极速查询。

#### 3.4 `database.py` — 数据库层
单表 `photos`，包含 24 个字段，覆盖文件信息、拍摄参数、GPS、标签和状态。使用 `aiosqlite` 进行异步访问。

#### 3.5 前端架构 (`index.js`)
纯 Vanilla JS 的单页面应用。
- **无限滚动**：利用 `IntersectionObserver` 懒加载分页数据。
- **移动端适配**：通过 CSS Transform 实现了抽屉式的滑动侧边栏。
- **Live Photo 交互**：鼠标悬停动态创建 `<video>` 标签覆盖原图进行播放。
- **层级筛选**：地点支持国家→城市层级点击，时间支持月份→具体日期筛选。
